Simple Trick To Debug Your CSS

css debugging web-development tricks

The Outline Trick

The fastest way to find misbehaving elements on the page is using the outline trick in CSS. This helps you find issues such as overflowing elements quickly, but more importantly it gives you a visual of the entire page.

You can use outline to set an outline on every element, which isn't going to affect the layout compared to using border â€” unless you specify box-sizing: border-box. 


* {
  outline: 1px solid tomato;
}

You can also add a background property, or use different outline colors for nested elements, if you want to make things more readable: 


* {
  outline: 1px solid tomato;
}

* * {
  outline: 1px solid black;
}

* * * {
  outline: 1px solid white;
}

Creating A Bookmarklet

You can turn this helper into a JavaScript bookmarklet, and use it when you need it. The only thing you have to do is create a regular bookmark in your browser, and include this code. 


javascript: (function () {
  const headElement = document.head;
  const styleElement = document.createElement('style');
  styleElement.setAttribute('debug-css', '');
  styleElement.innerText = '* { outline: 1px solid tomato; }';

  const debugElement = headElement.querySelector('[debug-css]');
  if (debugElement) return debugElement.remove();

  headElement.append(styleElement);
})();

The code uses an IIFE to create a <style> tag, append it to the <head> element, and checks if the element exists to toggle it.

Why This Method Works Better Than DevTools

While browser DevTools offer ways to inspect elements, this outline trick provides several advantages:

  1. It shows all elements at once, giving you a complete visual map of your page structure
  2. It reveals hidden overflow issues that might only appear in certain screen sizes
  3. It helps identify nesting problems and unexpected element relationships
  4. It works consistently across all browsers
  5. The bookmarklet makes it instantly available on any site you're working on

Extending The Technique

You can enhance this debugging approach with additional techniques: 


/* Highlight elements with overflow issues */
* {
  outline: 1px solid tomato;
  overflow: visible !important;
}

/* Identify z-index stacking contexts */
* {
  outline: 1px solid tomato;
  z-index: auto !important;
}

/* Debug flexbox layouts */
* {
  outline: 1px solid tomato;
}
[class*="flex"] {
  background-color: rgba(0, 100, 255, 0.1) !important;
}

These variations can help isolate specific types of layout problems when you're dealing with complex CSS issues.

Conclusion

The CSS outline debugging trick is one of those simple yet powerful techniques that should be in every web developer's toolkit. It takes just seconds to implement but can save hours of debugging time, especially when working with complicated layouts.

Next time you're scratching your head over a CSS layout issue, try adding this one line of code before diving into the browser's developer tools. You might be surprised at how quickly you can spot the problem!
